### States in Git

Состояния в Git
> * __untracked__ (неотслеживаемое)
> * __modified__ (измененное)
> * __staged__ (подготовленное)
> * __commited__ (закомиченное)
***

### Conventional Commits

Формат: `тип: глагол + что сделано`

#### Тип (фиксированный список — не придумывать свои)

- **feat** — добавил новую функциональность для пользователя
- **fix** — исправил баг (что-то было сломано в поведении)
- **docs** — изменения только в документации (README, SPEC, комментарии)
- **style** — форматирование кода (пробелы, отступы), без изменения логики
- **refactor** — переписал код без изменения внешнего поведения
- **perf** — изменения, улучшающие производительность
- **test** — добавил/изменил тесты
- **build** — изменения в системе сборки или зависимостях (gradle, npm)
- **ci** — изменения в конфигурации CI/CD
- **chore** — техническая рутина, не относящаяся к коду напрямую (конфиги, .gitignore)
- **revert** — отменяет предыдущий коммит

#### Глагол в описании (свободный, не фиксированный)

После типа идёт обычная фраза в повелительном наклонении:
add, update, remove, expand, fix (как глагол), rename, clean up...

Примеры:
- `feat: add password length validation`
- `docs: update README with usage instructions`
- `chore: add scratch directory to gitignore`

### Useful commands

> git clone "ссылка на репозиторий" — состояние untraced

> git branch branch-name — создает ветку с названием \
> git branch -a — показывает название веток + на какой находишься \
> git branch -v — посмотреть последний коммит на ветке \
> git branch -D branch-name — удалить ветку \
> git branch --move bad-branch-name corrected-branch-name — переименовать ветку 
 
> git checkout branch-name — перейдет на ветку \
> git checkout - — перейдет на предыдущую ветку \
> git checkout -b branch-name — создаст и перейдет на ветку

> git add . — переведет в состояние staged все файлы из состояния modified

> git commit -m "добавить понятные сообщения" — это основной объект в управлении контроля версий

> git init — Инициализирует новый Git репозиторий текущей директории \
> git push — Отправляет изменения из локального репозитория в удаленной \
> git pull — Обновляет текущею ветку с последней версией из удаленного репозитория \
> git merge — Сливает изменения указанной ветки в текущею ветку \
> git rebase — Переносит изменения на новую базу (чаще всего это другая ветка) \
> git config --list — проверить настройки (выход - :q)
